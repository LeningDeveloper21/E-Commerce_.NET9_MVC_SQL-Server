# Principios SOLID

## S - Responsabilidad Única
Una clase debe tener una sola razón para cambiar.

## O - Abierto/Cerrado
El código debe poder extenderse sin modificarse.

## L - Sustitución de Liskov
Una clase hija debe poder reemplazar a su padre sin romper el sistema.

## I - Segregación de Interfaces
Es mejor tener interfaces pequeñas y específicas.

## D - Inversión de Dependencias
Depender de abstracciones y no de implementaciones concretas.
